# 詳細設計（画面単位）

このフォルダには、サイトの **画面（ページ）ごとの詳細設計書** を置く。
要件定義（`docs/REQUIREMENTS.md`）と基本設計（`docs/BASIC_DESIGN.md`）を具体的な実装に落とすための資料。

## ファイル命名

- 画面単位で1ファイル
- **小文字 + ハイフン区切り**
- 例：`home.md` / `search.md` / `commands-index.md` / `reverse-lookup.md`

## 作り方

1. `SCREEN_TEMPLATE.md` をコピーして新規作成
2. 画面の目的とユーザーフローをまず確定
3. 表示項目・状態・イベント・エラーを埋める
4. 受け入れ条件（画面の完了定義）を書く

## 作成状況（MVP）

作成済み:

- `home.md`（トップ）
- `search.md`（検索）
- `commands.md`（コマンド一覧 A-Z）
- `reverse.md`（逆引き）

次に作成:

- `batch.md`（バッチ一覧）
- `topics.md`（トピック一覧）

補足:

- ページ実装が先に存在していても、受け入れ条件と状態遷移を合わせるために詳細設計を作成する。
- 命名は短く分かりやすいページ名を優先し、URLとの対応が分かる名前にする。
