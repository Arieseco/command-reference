# トップ（Home）

## 1. 目的

- ユーザーが **最短で目的の情報へ到達**できる（まず検索、次に主要導線）
- 「このサイトで何ができるか」を1画面で理解できる

## 2. 対象ユーザー / 利用シーン

- コマンド名をうろ覚えのまま探したい（例：`copy`/`xcopy`/`robocopy` の違い）
- 「用途」から探したい（例：ファイル操作、ネットワーク、環境変数）
- バッチ文法を確認したい（`for`、遅延展開、リダイレクト等）

## 3. 画面の場所（URL/導線）

- URL：`/`
- 主要導線：
  - ヘッダ検索 / トップの検索 → 検索結果（`/search/`）
  - トップ → コマンド一覧（A-Z）（`/commands/`）
  - トップ → 逆引き（用途）（`/reverse/`）
  - トップ → バッチ文法（`/batch/`）
  - トップ → topics（`/topics/`）

## 4. 画面構成（レイアウト）

React公式リファレンスに近い「検索最優先」の構成にしつつ、画面レイアウトは次の4分割とする。

- ヘッダ
- メイン
- サイドバー
- フッタ

### メインに置くもの（上から）

- ヒーロー（タイトル + サイト説明 + 大きい検索）
- クイックナビ（主要カテゴリのカード/ボタン）
- 逆引きの入口（用途カテゴリの一覧）

### サイドバーに置くもの

- コマンド一覧（A-Z）への導線
- 逆引き（用途）への導線
- バッチ文法/topics への導線
- よく使うタグのショートカット（**自動生成**）

## 5. 表示要素（コンポーネント）

UIは Material Design（公式）に寄せ、`material-web` を使用する。

| 要素 | 目的 | 表示条件 | 備考 |
| --- | --- | --- | --- |
| サイトタイトル | 認知 | 常に | クリックで`/`へ |
| グローバルナビ | 回遊 | 常に | `/commands/` `/reverse/` `/batch/` `/topics/` |
| ヒーロータイトル/説明 | 価値提示 | 常に | 1〜2行で簡潔に |
| 検索ボックス（大） | 目的達成 | 常に | `cmd`コマンド名/用途/オプションで検索 |
| 検索サジェスト | 発見性 | 検索入力時 | **必須**。コマンド名優先、別名も表示 |
| クイックカード（4〜6個） | 主要導線 | 常に | コマンド一覧/逆引き/バッチ文法/topics/検索の使い方 |
| 逆引き（用途）リスト | 目的導線 | 常に | タグ一覧から自動生成（要件） |
| サイドバー導線 | 回遊 | 常に | 主要ページ + 代表タグショートカット |

## 6. 状態（State）

- 初期状態：
  - ヒーローと検索が表示される
  - 逆引き（用途）リストはビルド生成データから表示（静的）
- ローディング：
  - 基本なし（静的）
  - 検索インデックスを初回ロードする方式の場合のみ、検索欄にローディング表示
- 結果あり：
  - トップでは検索結果は表示しない（Enter/検索ボタンで`/search/`へ遷移）
  - 入力中にサジェスト（候補）を表示
- 結果なし：
  - サジェストが空の場合は「該当なし。検索で全文を探す」を表示
- エラー：
  - 検索インデックスの読み込み失敗時は、検索欄付近にメッセージ + `/search/` への遷移ボタン

## 7. ユーザー操作とイベント

| 操作 | トリガ | 振る舞い | 備考 |
| --- | --- | --- | --- |
| 検索入力 | 入力 | サジェスト更新 | デバウンス：要検討 |
| Enterで検索 | Enter | `/search/?q=<query>` へ遷移 | クエリはURLエンコード |
| 検索ボタン | クリック | `/search/?q=<query>` へ遷移 | ボタン有無は要検討 |
| サジェスト選択 | クリック/Enter | 選択したページへ遷移 | 矢印キーで選択できる |
| カード選択 | クリック | 対応ページへ遷移 | |
| 用途カテゴリ選択 | クリック | `/reverse/?tag=<tag>` へ遷移 | reverseの実装に合わせる |

## 8. 検索/絞り込み仕様（該当する場合）

トップでの役割は「検索へ渡す入口」。

- 入力対象：コマンド名、用途語（例：コピー、圧縮）、オプション文字列（例：`/S`）
- サジェスト（必須）：
  - 候補数：最大10件（初期案）
  - 優先：コマンド名（完全一致/前方一致）
  - 次点：`aliases`（別名）
  - 表示：`title` + `summary` + 種別（command/batch/topic）
  - キーボード：↑↓で移動、Enterで決定、Escで閉じる

## 9. エラーハンドリング

- 検索関連のエラーは「検索できない」状態に落とすだけで、ページ閲覧自体は継続可能にする
- 表示文言は簡潔にし、復旧導線（再読み込み/検索ページへ）を用意する

## 10. アクセシビリティ（最低要件）

- Tab操作で検索→カード→用途カテゴリへ移動できる
- 検索サジェストを実装する場合、ARIAで読み上げ可能にする（候補リスト）
- フォーカスリングを消さない

## 11. 受け入れ条件（DoD）

- トップにアクセスすると **大きい検索ボックス** が見える
- 入力して Enter で **検索ページに遷移**できる（クエリがURLに反映される）
- コマンド一覧（A-Z）、逆引き、バッチ文法、topics に **1クリックで到達**できる
- 逆引き（用途）リストが表示される（静的データでOK）
