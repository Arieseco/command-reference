# 5xxエラー画面（サーバーエラー）

## 1. 目的

- サーバー側障害（5xx）発生時に、ユーザーへ明確な状況説明を行う
- ユーザーが再試行・トップ復帰・時間を置いた再アクセスを判断できるようにする

## 2. 対象ユーザー / 利用シーン

- GitHub Pages / CDN 側の一時障害でページ取得に失敗したとき
- デプロイ直後の伝播遅延や一時不整合で 5xx が返るとき
- ネットワーク経路問題で上流接続が不安定なとき

## 3. 画面の場所（URL/導線）

- URL：`/5xx/`（想定、将来実装）
- 主要導線：
  - 5xx 応答時のエラーページ表示
  - 手動確認用リンク（運用者向け）

## 4. 画面構成（レイアウト）

- メイン領域
  - エラーコード見出し（`500 / 502 / 503 / 504`）
  - 説明文（現在の状態と想定復旧）
  - 推奨操作ボタン
    - 再読み込み
    - トップへ戻る
    - 検索へ戻る（任意）
  - サポート文言（しばらく待って再試行）

## 5. 表示要素（コンポーネント）

| 要素 | 目的 | 表示条件 | 備考 |
| --- | --- | --- | --- |
| エラーコード表示 | 状態把握 | 常に | 例: `503 Service Unavailable` |
| 説明文 | 不安軽減 | 常に | 技術用語を避けて簡潔に |
| 再読み込みボタン | 復旧導線 | 常に | `location.reload()` |
| トップへ戻るボタン | 回遊復帰 | 常に | `withBase("/")` |
| 検索へ戻るリンク | 代替導線 | 任意 | `withBase("/search/")` |

## 6. 状態（State）

- 初期状態：
  - 5xxコードと説明文を表示
- ローディング：
  - なし（静的エラー画面）
- 復旧確認中：
  - 再読み込み後、通常ページへ遷移する可能性あり
- 継続障害：
  - 同じ 5xx 画面へ戻る

## 7. ユーザー操作とイベント

| 操作 | トリガ | 振る舞い | 備考 |
| --- | --- | --- | --- |
| 再読み込み | クリック | 現在URLを再取得 | 最短復旧確認 |
| トップへ戻る | クリック | `/` へ遷移 | ナビ再開 |
| 検索へ戻る | クリック | `/search/` へ遷移 | 代替導線 |

## 8. 検索/絞り込み仕様（該当する場合）

- なし（エラー画面のため検索機能は提供しない）

## 9. エラーハンドリング

- 想定対象コード：
  - `500 Internal Server Error`
  - `502 Bad Gateway`
  - `503 Service Unavailable`
  - `504 Gateway Timeout`
- 5xx詳細原因は画面に出しすぎず、運用ログで確認する
- 同時に `status.github.com` 等の外部ステータス確認導線を将来追加可能

## 10. アクセシビリティ（最低要件）

- 見出しで「エラー状態」を明確に伝える
- ボタンとリンクをキーボード操作で辿れる
- 色だけで状態を伝えない（文言で説明）

## 11. 受け入れ条件（DoD）

- 5xx時にユーザー向けエラーメッセージが表示される
- 再読み込みとトップ復帰の導線がある
- モバイル/PCで最低限読めるレイアウトを維持する
- 主要文言が日本語で統一される
