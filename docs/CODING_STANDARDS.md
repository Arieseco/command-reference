# コマンドプロンプトリファレンス：コーディング規約（初版）

このリポジトリは「コマンドプロンプト／バッチのリファレンス」を読みやすく・探しやすく保つための規約です。
迷ったら **読み手の検索性** と **再利用性（コピペで動く）** を優先します。

## ゴール

- **同じ種類のページは同じ型**（構成・見出し・表記）にそろえる
- **例はすべて実行可能**（必要なら前提条件を明記）
- **Windows環境差分**（cmd.exe / PowerShell / Windows 10/11、権限、文字コード）を明確にする

## ディレクトリ構成（docs）

- `docs/`
  - `commands/`：コマンドごとの解説（例：`copy.md`）
  - `batch/`：バッチ（`.bat`/`.cmd`）文法・テクニック
  - `topics/`：横断トピック（文字コード、リダイレクト、エスケープ等）
  - `assets/`：画像・図（必要になったら）

※最初は `docs/commands/` と `docs/batch/` だけでもOK。増えるたびにここに追記します。

## ファイル命名

- **小文字 + ハイフン区切り**：`setlocal.md` / `errorlevel.md` / `for-f.md`
- コマンド名そのものが記号を含む場合は、一般的な呼称に寄せる（例：`cmd-operators.md`）

## Markdown 方針

- 見出しは `##` から開始（ページタイトルは先頭の1行目に `#` を使ってもよいが、統一したい場合は `#` を使う）
- 箇条書きは `- ` を使う
- コードは **必ずコードブロック**（インラインで長いものを書かない）
- コードブロックの言語タグは、原則：
  - `bat`：バッチ例
  - `cmd`：cmd.exe での1行例（どちらでも良いが、混在は避ける）
  - `powershell`：PowerShell例
  - `text`：出力例

## frontmatter（必須）

- すべてのページは **frontmatterを必須** とする（検索/索引/逆引きのため）
- 基本キー（原則）：`title` / `type` / `tags` / `aliases` / `summary`
- `tags` と `aliases` は空配列（`[]`）でも良いが、キー自体は置く

## TODO運用（必須）

- `docs/TODO.md` などのチェックリストは、**完了した項目を必ず `- [x]` に更新**する
- 「やったのに未チェック」を禁止し、作業状況が一覧で分かる状態を維持する
- 作業終わりに、必ず `docs/TODO.md` を見直して **次にやるべきことを1つ決め**、会話の最後に **「次は〇〇をしますがよろしいですか？」** と確認する
- 作業途中で判断が必要になったら、放置せず **「これどうしますか？」** と質問して決める

## リンク（GitHub Pages `base` 対応）

- 内部リンク（`href` / `form action` など）は、原則 **`withBase()` を使って組み立てる**
  - GitHub Pages の `base: "/commanddictionary"` 配下でもリンク切れしないため

## 表記ルール（用語）

- **cmd.exe** と **コマンドプロンプト** は同義として扱う（文脈により使い分け）
- **バッチファイル**＝`.bat`/`.cmd`
- 重要な差分は明示：
  - `cmd.exe` 限定
  - バッチ（ファイル）限定
  - 対話（インタラクティブ）限定

## サンプル（例）の原則

- 例は「そのまま貼って動く」こと
- 依存がある場合は、例の直前に **前提条件** を箇条書きで書く
- 例は、可能なら **最小→実用** の順に2段階
- 可能なコマンドは **オプション（スイッチ）を使った例を最低1つ** 含める（既定動作との差が伝わるもの）
- 破壊的操作（削除・上書き・レジストリ変更・ネットワーク送信等）は：
  - 例の見出しに **危険** を明記
  - まず `echo` や `dry-run` 版を提示し、次に実行版を提示

## 文字コード・改行

- 文章（Markdown）は **UTF-8** を基本
- コマンド例は、必要な場合のみ文字コードに触れる（例：`chcp 65001`）
- 日本語の文字化けが絡む例は **出力例** も併記して再現性を上げる

## エラー・終了コードの扱い

- `%ERRORLEVEL%` / `errorlevel` の説明では、必ず：
  - 「**比較は `if errorlevel N` が “以上” 判定**」であることを明記
  - 例で意図が伝わるようにする

## リンクの方針

- 可能ならページ内リンクで解決（`topics/` を育てる）
- 外部リンクは最小限。貼るなら「何が書いてあるか」を一言添える

## テンプレート

新規ページ作成時は、次のテンプレをコピーして使います。

- コマンドページ：`docs/TEMPLATE_COMMAND.md`
- バッチ文法ページ：`docs/TEMPLATE_BATCH.md`
